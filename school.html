Signal Processing

This course gives an understanding on how we numerize a signal, what are the impacts and how we can elaborate a digital filter to filter an analog signal.

Keywords: Sound/Image processing, Laplace/Fourier, Filters, Analog/Discrete signals, Sampling

1. Discrete Fourier Transform (DFT) and Analog Filter Synthesis

With a view to processing digital signals, we need to turn a continuous (or analog) signal into a discrete signal that consists of a finite number of samples that can be handled by a processor. Therefore, we use the DFT, let's remind us quickly of that.

  1. Sampling & Quantification: turns the continuous signal into a discrete signal.
  2. Temporal truncation of the signal: get an finite amount of samplings.
  3. Spectral discretization: turns the continuous spectrum into a discrete spectrum.
  4. Zero-Padding (optional): increase the spectrum's resolution.

However, each one of these steps has an impact on the signal. Indeed, discretizing a signal is not without consequences and some parameters should be chosen wisely to avoid aliasing and other artifacts.

2. Digital Filter Synthesis

There exists 2 types of digital filters: IIR (Infinite Impulse Response) and FIR (Finite Impulse Response).

  • IIR: the output depends on the current and previous inputs and outputs > \( s[n] = \sum_{i=0}^{M-1}{a_i e[i]} + \sum_{j=0}^{N-1}{a_i s[n-i]} \)

    • The purpose of this method is to determine the equivalent digital form \( H(z) \) (\(z\)-domain) of the analog template filter \( H(p) \). From there, we can deduce the recursive algorithm \( s[n] \) that will implement the digital filter. This method is more accurate and will be generally preferred over the RIF method. However, the analog filter must meet some requirements to be feasible (stability).

    - Invariance of the Impulse Response: This method consists of starting with an analog template

    - Discretization of the inverse DFT:

    All in all, this method has some benefits - more selective, faster and requires only a few memory registers - and some drawbacks - can be unstable (poles), non-linear phase.

  • FIR: the output depends only on the current and previous inputs > \( s[n] = \sum_{i=0}^{N-1}{a_i e[i] h[n-i]} \)

    • The purpose of this method is to determine an approximation of the filter's impulse response and then convolute it with the input signal. As we consider only an approximation, the filter's characteristics get modified and we have to deal with some drawbacks such as the apparition of wavelets in the filter's spectrum (caused by the truncation --> convolution with sinc).

    - Coherence of the Impulse Response:

    1. Sampling frequency: Determine \( f_{max}\) and apply the Shannon theorem: \(f_e\ \geq\ 2f_{max}\)
    2. Impulse response (IR): Find the impulse response of the analog filter.
    3. Signal truncation: Truncate the IR after a rank N such that \( h(N) \geq h_{max}/100\)

    - Discretization of the inverse DFT: We determine the IR samples using direclty the inverse DFT definition > \( h[n] = \frac{1}{N} \sum_{k=-\frac{N}{2}}^{\frac{N}{2}-1} X(k\delta f) e^{j2\pi \frac{n}{N}k} \).

    All in all, this method has some benefits - always stable (no poles), linear phase (if symmetric) - and some drawbacks - less selective, less efficient and requires more memory space.

1. Basis

In this first lab session, we basically covered :

  1. the difference between hubs and switchs;

    2. Hubs are dumb, they broadcast all messages, therfore it is very easy to eavesdrop the network traffic and it reduces the bandwith. Switchs are smart, they only broadcast the messages to the targeted host, therefore it is more secure and it increases the bandwith.

  2. how to configure a network interface;
  3. basic protocols such as ICMP and ARP (MAC & IP addresses), ncat tool;

    4. ICMP is a protocol implemented on the network (3rd) layer where the IP address lies. ARP is a protocol that matches an MAC (datalink layer) with an IP address (network layer) within an ARP table. This protocol is used when a host sends data to an IP address

  4. OSI and TCP/IP models, encapsulation of data, subnet-mask.

2. Building a more sophisticated network

In this lab session, we set up a network of PCs that consisted of several switchs :

  1. reachability, hierarchial network structure;

    2. Each host on a sub-network has a default route that represents an access point to send a packet outside.

  2. IP forwarding & translation , routing table;

    3. When a packet is sent to a router, the dest MAC is the router's and the dest IP is the targeted host on the network. When it is forwarded by the router, the src MAC is the router's and the src IP initial host's.

  3. TTL (Time To Live)

    4. When a packet is sent, the TTL is set to 64. Each time the packet is forwarded, the TTL is decremented by 1. If the TTL reaches 0, the packet is dropped and an ICMP message is sent to the initial host.
